# Contributing to SAOGEN SAO

Thank you for your interest in contributing to SAOGEN SAO.

SAOGEN is being developed as a Symbiotic Autonomous Organization: a coordination layer for builders, AI systems, researchers, token holders, and specialized development groups working toward shared technological progress. Contributions are welcome from developers, designers, researchers, writers, community moderators, infrastructure operators, and strategic partners.

This document defines the official contribution process for the SAOGEN repository and related projects.

---

## 1. Contribution Philosophy

SAOGEN is not only a software project. It is a long-term framework for coordinated innovation, shared intellectual property, and AI-assisted development.

Contributors are encouraged to work with the following principles:

- Build for long-term resilience, not short-term hype.
- Prefer clarity, simplicity, and maintainability.
- Respect the distinction between ideas, prototypes, and production-ready systems.
- Communicate openly and constructively.
- Treat AI-assisted development as a legitimate part of the creative and engineering process.
- Avoid misleading claims, financial promises, or exaggerated guarantees.
- Help make SAOGEN useful for real builders and real communities.

## 3. Contribution Process

### Step 1: Discuss the Idea

Before starting major work, open a GitHub Issue or discuss the proposal in the official SAOGEN community channel.

For small fixes, such as typos or minor layout improvements, you may open a pull request directly.

### Step 2: Fork the Repository

Fork the repository to your own GitHub account and create a new branch:

```bash
git checkout -b feature/your-feature-name
```

Use clear branch names, for example:

```bash
feature/homepage-redesign
fix/mobile-menu
content/update-about-page
research/sao-governance-notes
```

### Step 3: Make Your Changes

Keep changes focused and easy to review. Avoid combining unrelated changes in one pull request.

Recommended commit style:

```bash
git commit -m "Add homepage hero section"
git commit -m "Fix mobile navigation layout"
git commit -m "Update SAO governance description"
```

### Step 4: Test Locally

Before submitting, test your changes locally when possible.

For a typical React/Vite-style frontend:

```bash
npm install
npm run dev
npm run build
```

Do not submit broken builds unless the pull request clearly states that it is experimental or requires help.

### Step 5: Open a Pull Request

Open a pull request against the main SAOGEN repository.

Your pull request should include:

- A short summary of what changed
- Why the change is useful
- Screenshots for visual changes, if applicable
- Any known limitations or unfinished parts

---

## 4. Pull Request Review Standards

Pull requests may be reviewed for:

- Code quality
- Security risk
- Maintainability
- Alignment with SAOGEN goals
- Accuracy of public claims
- UI/UX quality
- Compatibility with existing architecture
- Legal or reputational risk

SAOGEN maintainers may request changes before merging.

Not every contribution is guaranteed to be accepted, but all serious contributions should be treated respectfully.

---

## 5. AI-Assisted Contributions

SAOGEN welcomes AI-assisted development.

Contributors may use tools such as ChatGPT, Codex, Grok, Claude, Gemini, or other AI systems to help with:

- Code generation
- Documentation
- Research
- Refactoring
- Testing ideas
- UI/UX drafts
- Governance design

However, contributors remain responsible for verifying their submissions.

Do not submit AI-generated content blindly. Review it for accuracy, security, originality, and project alignment.

---

## 6. Security Policy

Do not publicly disclose serious vulnerabilities before maintainers have time to respond.

If you discover a security issue, contact the maintainers privately first. Include:

- A clear description of the issue
- Steps to reproduce it
- Potential impact
- Suggested fix, if known

Never submit secrets, private keys, API tokens, wallet seed phrases, database credentials, or production passwords to the repository.

---

## 7. Legal and Financial Communication

SAOGEN may involve blockchain, governance, shared IP, and future R&D funding models. Because of this, public wording must be careful.

Contributors should avoid:

- Promising profits
- Guaranteeing token value
- Making investment claims
- Presenting speculative ideas as completed products
- Claiming official partnerships without written confirmation
- Making legal, tax, or financial guarantees

Use precise language. Separate vision, research, prototype, and production status clearly.

---

## 8. Code Style

General expectations:

- Keep code readable and modular.
- Use meaningful names for files, functions, and components.
- Avoid unnecessary complexity.
- Prefer reusable components.
- Keep public-facing text clear and professional.
- Comment only where it improves understanding.

If a project-specific formatter or linter is added later, contributors should follow it.

---

## 9. Community Standards

SAOGEN contributors are expected to communicate with respect and intellectual honesty.

Healthy disagreement is welcome. Personal attacks, harassment, spam, and bad-faith behavior are not welcome.

The goal is to create an environment where different experiences, technical skills, and visions can complement each other.

---

## 10. Recognition

Meaningful contributors may be recognized in project documentation, release notes, contributor lists, Discord roles, or future governance structures.

Recognition does not automatically imply employment, token allocation, legal partnership, or ownership rights unless separately agreed in writing.

---

## 11. License and Ownership

Unless otherwise stated in a separate written agreement, contributions submitted to the SAOGEN repository are provided under the repository's license.

By opening a pull request, you confirm that you have the right to submit the contribution and that it does not knowingly violate third-party rights.

For major strategic, research, or intellectual property contributions, contributors may request a separate discussion before submitting the work publicly.

---

## 12. Contact

For contribution discussions, use the official SAOGEN GitHub Issues, Pull Requests, or community channels.

SAOGEN welcomes builders who want to help create intelligent, scalable, and long-term systems through human-AI collaboration.
